### AI 生成内容的问题

初代 Prompt 没有把评分点说清楚，容易引出三类不合规结果：任务2可能只使用 pandas 条件筛选而没有实质使用 NumPy；任务3可能修改题目规定的函数签名，或画出默认置信区间而非标准差；任务4可能主观假定 8:00~9:00 为高峰小时，没有先对 24 小时计数后自动识别。

### 优化后的 Prompt

这次优化将“用哪个库”、“函数接口”、“统计口径”和“验收输出”同时写清楚，使结果能够直接对照评分表检查。

我自己修改的地方：我没有直接保留 AI 给出的单一 seaborn 写法，而是对新版 `errorbar='sd'` 和旧版 `ci='sd'` 做了兼容；另外增加了 CSV 存在性、必要字段和输出文件数量检查。每一张图生成后都打开检查，而不是只看程序没有报错。

## 3. Debug 记录

### 现象

线路条形图需要展示标准差，但 seaborn 新旧版本的参数不一致：旧版使用 `ci='sd'`，0.12 及更新版本则推荐 `errorbar='sd'`。如果只保留一种写法，可能在老师的环境中报错。

### 解决过程

我先用当前 seaborn 0.13 环境运行 `errorbar='sd'`，确认图中误差棒实际出现；然后在调用外层加入 `try/except`，当新参数不受支持时退回 `ci='sd'`。两个分支都显式传入 `capsize=0.3`，同时将 Top15 线路顺序由 `analyze_route_stops` 返回结果控制，避免绘图后名次被重新排列。最后查看输出 PNG，确认标签、柱体和误差棒没有裁切。

补充测试：我又模拟了“CSV 路径不存在”和“CSV 缺少必要字段”两种情况，程序会给出明确的文件或字段错误，不会在后续聚合时才出现难以定位的异常。重复运行时，任务5会先清理上一次生成的 txt，再重新导出 1101~1120，避免旧文件残留。

## 4. 人工代码审查（任务4 PHF 核心代码）

下面这段是我重点逐行检查的 PHF 实现。审查时我特别核对了时间索引、重采样窗口和公式中 12/4 两个系数。

```python
# argmax 返回刷卡量最大值的小时索引，不手工预设早高峰。
peak_hour = int(np.argmax(hourly_counts))
# 只保留高峰小时中的上车刷卡记录。
peak_records = boardings.loc[boardings["hour"].eq(peak_hour)].copy()
# 记录条数是两个 PHF 公式的共同分子。
peak_volume = int(len(peak_records))
# 设置并排序时间索引，保证 resample 能够按自然时间窗口分组。
peak_timeline = peak_records.set_index("交易时间").sort_index()

# 将高峰小时分成12个5分钟窗口，计算每个窗口的刷卡量。
counts_5min = peak_timeline.resample("5min").size()
# 取出最大5分钟窗口的刷卡量。
max_5_volume = int(counts_5min.max())
# PHF5 分母用 12 个5分钟窗口乘以最大窗口流量。
phf5 = peak_volume / (12 * max_5_volume)

# 将高峰小时分成4个15分钟窗口，计算每个窗口的刷卡量。
counts_15min = peak_timeline.resample("15min").size()
# 取出最大15分钟窗口的刷卡量。
max_15_volume = int(counts_15min.max())
# PHF15 分母用 4 个15分钟窗口乘以最大窗口流量。
phf15 = peak_volume / (4 * max_15_volume)
```

在本数据集上，程序自动识别的高峰小时为 09:00~10:00，刷卡量为 14961 次；最大5分钟刷卡量为 1424 次，`PHF5=0.8755`；最大15分钟刷卡量为 4060 次，`PHF15=0.9212`。两个系数都小于 1 且接近 1，数值范围符合 PHF 定义。
