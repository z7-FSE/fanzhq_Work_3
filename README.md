## 3. Debug 记录

### 现象

线路条形图要显示标准差，但 seaborn 的新旧版本参数不一样；旧版用 `ci='sd'`，0.12 及后面的版本用 `errorbar='sd'`，如果代码只写一种，换个版本运行就有可能报错。

### 解决过程

先在当前 seaborn 0.13 环境中运行 `errorbar='sd'`，生成的图里能够看到误差棒；之后在调用外面加了 `try/except`，新参数不能使用时，程序就改用 `ci='sd'`；两种写法都传入 `capsize=0.3`，Top15 的线路顺序则根据 `analyze_route_stops` 返回的结果排列。图片保存后再打开看了一遍，标签、柱子和误差棒没有被截掉。

程序还测试了 CSV 路径不存在、缺少必要字段和重复运行这三种情况；重复运行时，任务5先删除旧 txt，然后重新导出 1101~1120。

## 4. 人工代码审查（任务4 PHF 核心代码）

任务4中计算 PHF 的主要代码如下：

```python
# argmax 返回刷卡量最大值的小时索引，不手工预设早高峰。
peak_hour = int(np.argmax(hourly_counts))
# 只保留高峰小时中的上车刷卡记录。
peak_records = boardings.loc[boardings["hour"].eq(peak_hour)].copy()
# 记录条数是两个 PHF 公式的共同分子。
peak_volume = int(len(peak_records))
# 设置并排序时间索引，保证 resample 能够按自然时间窗口分组。
peak_timeline = peak_records.set_index("交易时间").sort_index()

# 将高峰小时分成12个5分钟窗口，计算每个窗口的刷卡量。
counts_5min = peak_timeline.resample("5min").size()
# 取出最大5分钟窗口的刷卡量。
max_5_volume = int(counts_5min.max())
# PHF5 分母用 12 个5分钟窗口乘以最大窗口流量。
phf5 = peak_volume / (12 * max_5_volume)

# 将高峰小时分成4个15分钟窗口，计算每个窗口的刷卡量。
counts_15min = peak_timeline.resample("15min").size()
# 取出最大15分钟窗口的刷卡量。
max_15_volume = int(counts_15min.max())
# PHF15 分母用 4 个15分钟窗口乘以最大窗口流量。
phf15 = peak_volume / (4 * max_15_volume)
```

程序在这份数据中找到的高峰小时是 09:00~10:00，这一小时的刷卡量为 14961 次；最大5分钟刷卡量是 1424 次，所以 `PHF5=0.8755`；最大15分钟刷卡量是 4060 次，所以 `PHF15=0.9212`。两个结果都小于 1，同时也比较接近 1，结果在 PHF 的取值范围内。
